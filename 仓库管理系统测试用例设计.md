| 5.8 | 取消草稿状态的盘库单 | 正向 | 员工登录，存在草稿状态的盘库单 | 1. 打开草稿盘库单<br>2. 点击取消 | 取消成功，盘库单状态变为已取消 |

## 测试执行说明

### 环境准备
1. 确保MongoDB服务已启动
2. 后端服务运行在 http://localhost:3000
3. 已创建测试用户：admin/123456（管理员）、manager/123456（经理）、staff1/123456（员工）、staff2/123456（员工）
4. 已安装测试依赖：`npm install supertest jest --save-dev`

### 执行命令
```bash
# 运行所有测试用例
npm test

# 运行测试并查看报告
npm run test:report
```

### 测试结果说明
- 所有测试用例执行完成后会自动生成《仓库管理系统完整测试报告.md》
- 报告包含测试概览、bug列表、详细测试结果
- 通过率低于90%时需要修复bug后重新测试
