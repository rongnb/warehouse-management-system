## Bug 修复建议

### Bug 1: 供应商手机号格式未校验
**问题描述**: 后端接口未对供应商手机号进行格式校验，错误格式的手机号可以成功保存
**修复方案**: 在供应商模型(Supplier.js)中添加手机号正则校验：
```javascript
phone: {
  type: String,
  required: true,
  match: [/^1[3-9]\d{9}$/, '请输入正确的手机号格式']
}
```

### Bug 2: 已审核入库单可重复审核
**问题描述**: 已审核状态的入库单可以再次提交审核，导致状态被错误更新
**修复方案**: 在入库审核接口中添加状态校验，已审核/已驳回的单据不能再次审核：
```javascript
if (transaction.status !== 'pending') {
  return res.status(400).json({ message: '该单据已审核，不能重复审核' });
}
```
