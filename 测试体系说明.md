# 仓库管理系统自动化测试体系说明

## 🎯 测试体系架构
本测试体系覆盖单元测试、集成测试、E2E测试三个层级，实现登录、盘库全流程、权限控制等核心功能的全量覆盖。

```
warehouse-management/
├── test/
│   ├── unit/                     # 单元测试
│   │   ├── auth.test.js          # 登录认证模块测试
│   │   └── permissions.test.js   # 权限控制模块测试
│   ├── integration/              # 集成测试
│   │   └── stocktake.test.js     # 盘库全流程集成测试
│   ├── e2e/                      # E2E端到端测试
│   │   ├── login.test.js         # 登录页面E2E测试
│   │   └── stocktake.test.js     # 盘库流程E2E测试
│   ├── setup.js                  # Jest测试环境配置
│   ├── setupTestEnv.js           # 测试环境变量配置
│   └── automated_test_runner.js  # 自动化测试执行器
├── jest.config.js                # Jest配置
├── playwright.config.js          # Playwright E2E测试配置
└── package.json                  # 测试脚本配置
```

## 🚀 测试执行方式

### 方式一：一键自动化测试（推荐）
自动执行所有测试、检测并修复Bug、生成完整测试报告：
```bash
npm run test:automated
```

### 方式二：分类执行测试
```bash
# 运行所有单元测试
npm run test:unit

# 运行所有集成测试
npm run test:integration

# 运行测试并生成覆盖率报告
npm run test:coverage

# 运行E2E端到端测试
npm run test:e2e

# 运行所有测试（单元+集成+E2E）
npm run test:all
```

## 📊 测试报告内容
自动化测试执行完成后会自动生成`测试报告.md`，包含：
- 测试基本信息（时间、环境）
- 测试概览（总用例数、通过数、失败数、通过率）
- 测试覆盖率（语句、分支、函数、行覆盖率）
- 各测试套件详细结果
- 发现的Bug列表
- 已修复的问题列表
- 测试结论

## 🛠️ 测试特性
1. **真实有效**：所有测试用例基于实际业务流程设计，使用真实测试数据
2. **自动修复**：测试过程中发现的常见Bug可以自动定位并修复
3. **自动重测**：修复完成后自动重新运行失败的测试用例验证修复效果
4. **覆盖率统计**：自动统计代码覆盖率，确保核心功能全覆盖
5. **环境隔离**：单元测试和集成测试使用内存数据库，不影响生产数据

## 🎯 测试覆盖率目标
- 核心业务代码覆盖率 ≥ 80%
- 登录、盘库、权限控制核心模块覆盖率 ≥ 90%

## 📝 测试环境要求
- Node.js ≥ 16.0.0
- MongoDB ≥ 4.0（或使用内置内存MongoDB）
- 后端服务运行在3000端口
- 前端服务运行在5176端口
