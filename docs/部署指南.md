# 部署指南

## 📋 环境要求

### 基础环境
- Node.js >= 16.0.0
- MongoDB >= 4.0
- npm 或 yarn 包管理器

### Docker 部署（推荐
- Docker >= 20.0.0
- Docker Compose >= 2.0.0

---

## 🚀 快速部署（推荐：Docker Compose 一键部署

### 1. 克隆项目
```bash
git clone <仓库地址>
cd warehouse-management
```

### 2. 一键启动
```bash
docker-compose up -d
```

### 3. 访问系统
- 前端地址：http://localhost
- 默认账号：
  - 管理员：admin / 123456
  - 仓管员A：keeper_a / 123456
  - 仓管员B：keeper_b / 123456

### 4. 停止服务
```bash
docker-compose down
```

### 5. 查看日志
```bash
# 查看所有服务日志
docker-compose logs -f

# 查看后端日志
docker-compose logs -f backend

# 查看前端日志
docker-compose logs -f frontend

# 查看数据库日志
docker-compose logs -f mongodb
```

## 🛠️ 手动部署

### 1. 安装依赖

#### 后端依赖安装
```bash
cd backend
npm install
```

#### 前端依赖安装
```bash
cd ../frontend
npm install
```

### 2. 数据库初始化

#### 方式一：自动初始化（推荐）
```bash
cd ../backend
npm run init-db
```

#### 方式二：手动执行初始化脚本
##### Linux/macOS
```bash
cd ../database
chmod +x init.sh
./init.sh
```

##### Windows
```bash
cd ../database
init.bat
```

### 3. 配置环境变量

#### 后端配置（backend/.env）
```env
PORT=3000
MONGODB_URI=mongodb://localhost:27017/warehouse
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d
UPLOAD_PATH=./uploads
```

### 4. 启动服务

#### 方式一：一键启动脚本
##### Linux/macOS
```bash
chmod +x start.sh
./start.sh
```

##### Windows
```bash
start.bat
```

#### 方式二：手动分别启动

##### 启动后端
```bash
cd backend
npm run dev
```
后端服务运行在 http://localhost:3000

##### 启动前端
```bash
cd frontend
npm run dev
```
前端服务运行在 http://localhost:5173

### 5. 生产部署

#### 前端构建
```bash
cd frontend
npm run build
```

#### 使用 Nginx 配置示例
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### 使用 PM2 管理后端进程
```bash
npm install -g pm2
cd backend
pm2 start server.js --name warehouse-backend
```

## 🔧 配置说明

### 数据库配置
- 默认数据库名称：warehouse
- 默认端口：27017
- 如需修改数据库连接地址，请修改 `backend/.env` 中的 `MONGODB_URI` 参数

### JWT 配置
- `JWT_SECRET`：JWT 加密密钥，生产环境请务必修改为复杂的随机字符串
- `JWT_EXPIRES_IN`：Token 过期时间，默认 7 天

### 文件上传配置
- `UPLOAD_PATH`：文件上传存储路径，默认 `./uploads`

## 📦 数据备份与恢复

### 数据备份
```bash
mongodump -d warehouse -o backup/
```

### 数据恢复
```bash
mongorestore -d warehouse backup/warehouse
```

### Docker 环境备份
```bash
# 备份数据库
docker exec warehouse-mongodb mongodump -d warehouse -o /data/backup
docker cp warehouse-mongodb:/data/backup ./backup

# 恢复数据库
docker cp ./backup warehouse-mongodb:/data/backup
docker exec warehouse-mongodb mongorestore -d warehouse /data/backup/warehouse
```
