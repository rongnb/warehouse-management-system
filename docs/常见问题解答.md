# 常见问题解答

## ❓ 登录相关问题

### Q1：登录时提示「用户名或密码错误」怎么办？
**A：**
1. 首先确认输入的用户名和密码是否正确，注意大小写
2. 如果忘记密码，请联系管理员重置密码
3. 管理员可以在「用户管理」页面重置用户密码

### Q2：登录后页面显示空白或报错怎么办？
**A：**
1. 请清除浏览器缓存后重新尝试
2. 检查网络连接是否正常
3. 确认后端服务是否正常运行
4. 尝试使用其他浏览器登录（推荐使用 Chrome、Edge、Firefox）

### Q3：登录后一段时间自动退出怎么办？
**A：**
1. 系统默认 Token 有效期为 7 天，超时后会自动退出
2. 这是正常的安全机制，重新登录即可
3. 如果频繁出现自动退出，请检查系统时间是否正确

## ❓ 盘库相关问题

### Q4：发起盘库时看不到仓库选项怎么办？
**A：**
1. 请确认您的账号是否有仓管员或管理员权限
2. 检查数据库中是否已初始化仓库数据
3. 可以尝试重新登录刷新权限
4. 如果仍有问题，请联系管理员检查数据配置

### Q5：录入盘点数据时无法保存怎么办？
**A：**
1. 检查网络连接是否正常
2. 确认所有必填字段是否都已填写
3. 实际库存数量不能为负数，请检查输入值
4. 如果仍无法保存，请刷新页面后重新尝试

### Q6：为什么我看不到「复核」按钮？
**A：**
1. 盘库录入和复核必须由不同的用户操作，同一用户不能复核自己录入的盘库数据
2. 请确认您不是该盘库任务的录入人
3. 确认您的账号是否有复核权限（仓管员或管理员都可以复核）
4. 确认盘库状态是否为「待复核」

### Q7：盘库复核不通过后怎么办？
**A：**
1. 复核驳回后，盘库状态会变为「待录入」
2. 原录入人可以重新编辑盘点数据
3. 修改完成后再次提交等待复核
4. 建议仔细核对数据后再提交，避免反复驳回

### Q8：盘库完成后可以修改数据吗？
**A：**
1. 盘库复核通过后，数据将无法修改
2. 如果确实需要修改，需要重新发起新的盘库任务
3. 建议在复核阶段仔细核对数据，确保准确性

## ❓ 系统配置问题

### Q9：如何修改盘库频率？
**A：**
1. 只有管理员账号有权限修改盘库频率
2. 登录管理员账号，进入「系统设置」页面
3. 在「盘库配置」中选择需要的频率（月度/季度/半年度/年度）
4. 保存后立即生效

### Q10：如何新增用户账号？
**A：**
1. 使用管理员账号登录
2. 进入「用户管理」页面
3. 点击「新增用户」按钮
4. 填写用户信息，分配角色权限
5. 保存后新用户即可使用

### Q11：如何修改用户密码？
**A：**
- **用户自己修改**：登录后进入「个人中心」，选择修改密码
- **管理员修改**：进入「用户管理」页面，找到对应用户，点击「重置密码」

## ❓ 部署相关问题

### Q12：Docker 部署时启动失败怎么办？
**A：**
1. 检查端口是否被占用：80、3000、27017 端口是否被其他服务占用
2. 查看服务日志定位问题：`docker-compose logs -f`
3. 确认 Docker 和 Docker Compose 版本是否符合要求
4. 尝试重新构建镜像：`docker-compose build --no-cache`

### Q13：服务启动后前端无法连接后端怎么办？
**A：**
1. 检查后端服务是否正常启动：访问 http://localhost:3000/health 看是否返回正常
2. 检查前端代理配置是否正确
3. 确认防火墙是否开放了 3000 端口
4. 查看浏览器控制台报错信息，定位具体问题

### Q14：数据库连接失败怎么办？
**A：**
1. 确认 MongoDB 服务是否正常启动
2. 检查 `backend/.env` 中的 `MONGODB_URI` 配置是否正确
3. 确认 MongoDB 是否开启了远程访问权限
4. 检查防火墙是否开放了 27017 端口

### Q15：如何备份数据库数据？
**A：**
- **手动部署**：执行 `mongodump -d warehouse -o backup/` 进行备份
- **Docker 部署**：执行 `docker exec warehouse-mongodb mongodump -d warehouse -o /data/backup`

### Q16：升级系统版本时如何保留数据？
**A：**
1. 升级前务必先备份数据库
2. Docker 部署时，数据库数据存在 Docker 卷中，重启或重建容器不会丢失数据
3. 手动部署时，只要不删除 MongoDB 数据目录，数据就不会丢失
4. 升级后建议测试所有功能是否正常

## ❓ 其他问题

### Q17：系统支持移动端访问吗？
**A：**
- 系统采用响应式设计，支持手机、平板等移动端设备访问
- 建议使用手机浏览器访问，操作体验和 PC 端一致

### Q18：导出的报表在哪里查看？
**A：**
- 导出的文件会自动下载到浏览器默认下载目录
- 可以在浏览器的下载记录中找到导出的文件
- 导出格式为 Excel，需要使用 Office 或 WPS 等软件打开

### Q19：系统运行缓慢怎么办？
**A：**
1. 检查服务器 CPU、内存、磁盘使用率是否过高
2. 清理浏览器缓存和 Cookie
3. 检查网络连接速度是否正常
4. 如果数据量过大，可以联系管理员优化数据库索引

### Q20：发现系统 Bug 或有功能建议怎么办？
**A：**
1. 请记录 Bug 复现步骤和报错信息
2. 联系系统管理员或开发人员反馈问题
3. 功能建议也可以直接反馈给管理员，我们会根据需求进行评估开发
