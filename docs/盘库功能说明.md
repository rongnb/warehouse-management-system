# 盘库功能说明文档

## 功能概述
盘库功能是仓库管理系统的核心功能之一，支持双人核实的盘库流程，自动计算盘盈盘亏数据，生成盘库报表，并永久留存盘库记录。

## 主要功能
### 1. 双人核实盘库流程
- **流程步骤**：
  1. 创建盘库单（草稿状态）
  2. 录入实际库存数据
  3. 提交盘库单（进入核实状态）
  4. 第一核实人确认
  5. 第二核实人确认（需要manager或admin权限，且不能与第一核实人相同）
  6. 盘库完成，自动更新库存

- **权限控制**：
  - 普通员工：可以创建、编辑、提交盘库单，作为第一核实人
  - 经理/管理员：可以作为第二核实人，完成盘库流程

### 2. 自动计算盘盈盘亏
- 系统自动对比系统库存与实际库存，计算每个商品的差异
- 自动统计总盘盈数量、总盘盈金额、总盘亏数量、总盘亏金额
- 差异类型自动识别：盘盈（实际>系统）、盘亏（实际<系统）、无差异

### 3. 盘库记录管理
- 盘库记录永久保存，支持多条件查询：
  - 按盘点单号/标题模糊搜索
  - 按状态筛选（草稿、核实中、已完成、已取消）
  - 按仓库筛选
  - 按创建时间范围筛选
- 支持分页显示

### 4. 盘库报表导出
- 已完成的盘库单支持导出为Excel报表
- 报表包含：
  - 盘库基本信息（单号、标题、仓库、时间、核实人等）
  - 汇总统计信息（盘盈盘亏总览）
  - 明细数据（每个商品的盘点详情）

## 接口说明
### 盘库相关接口
| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/stocktake` | GET | 获取盘库列表 | 所有登录用户 |
| `/api/stocktake/:id` | GET | 获取盘库详情 | 所有登录用户 |
| `/api/stocktake` | POST | 创建盘库单 | 所有登录用户 |
| `/api/stocktake/:id` | PUT | 更新盘库单（录入实际库存） | 所有登录用户 |
| `/api/stocktake/:id/submit` | POST | 提交盘库单 | 所有登录用户 |
| `/api/stocktake/:id/confirm` | POST | 核实盘库单 | 所有登录用户（第二核实需要manager/admin） |
| `/api/stocktake/:id/cancel` | POST | 取消盘库单 | 所有登录用户 |
| `/api/stocktake/:id/export` | GET | 导出盘库报表 | 所有登录用户 |

## 使用说明
### 1. 创建盘点单
1. 进入"库存盘点"页面，点击"新建盘点"按钮
2. 填写盘点标题，选择盘点仓库，填写备注（可选）
3. 点击"保存"，系统自动拉取该仓库所有商品的当前库存作为系统库存

### 2. 录入实际库存
1. 在盘点明细中，逐个录入商品的实际库存数量
2. 系统自动计算差异和差异类型
3. 可以填写每个商品的盘点备注
4. 点击"保存"更新盘点单

### 3. 提交审核
1. 确认所有实际库存录入完成后，点击"提交审核"
2. 盘点单状态变为"核实中"，无法再编辑

### 4. 双人核实
1. **第一核实人**：进入盘点详情，点击"核实"按钮，填写核实意见，确认提交
2. **第二核实人**：需要经理或管理员权限，且不能与第一核实人相同，进入盘点详情，点击"核实"按钮，填写核实意见，确认提交
3. 第二核实人确认后，盘点完成，系统自动更新库存，并生成对应的出入库记录

### 5. 导出报表
1. 已完成的盘点单，点击"导出"按钮
2. 系统自动生成Excel报表并下载

### 6. 取消盘点单
1. 未完成的盘点单可以点击"取消"按钮
2. 填写取消原因，确认后盘点单状态变为"已取消"

## 注意事项
1. 盘库完成后，库存会自动更新，同时生成盘盈/盘亏的出入库记录
2. 已完成的盘库单无法修改或取消
3. 第二核实人必须具备经理或管理员权限，且不能与第一核实人是同一用户
4. 盘库记录永久保存，支持随时查询和导出
