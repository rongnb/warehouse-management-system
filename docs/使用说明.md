# 使用说明

## 🌟 功能概述

本仓库管理系统仅保留核心的**登录功能**和**盘库功能**，盘库采用双人核实机制，默认按季度进行盘库。

---

## 🔐 用户登录

### 访问地址
- 本地部署：http://localhost:5173 或 http://localhost
- 服务器部署：http://your-domain.com

### 默认账号
| 角色 | 账号 | 密码 | 权限说明 |
|------|------|------|----------|
| 系统管理员 | `admin` | `123456` | 系统配置、用户管理、盘库复核、数据查看 |
| 仓管员A | `keeper_a` | `123456` | 发起盘库、录入盘点数据 |
| 仓管员B | `keeper_b` | `123456` | 盘库复核、录入盘点数据 |

### 登录流程
1. 打开系统登录页面
2. 输入账号和密码
3. 点击「登录」按钮进入系统
4. 登录成功后，系统会根据用户角色显示对应的功能菜单

## 📋 盘库功能

### 盘库规则
- **双人核实**：盘库需要两个不同的仓管员操作，一人发起并录入数据，另一人复核确认
- **盘库频率**：默认每季度一次，管理员可在系统设置中调整
- **盘库范围**：支持按仓库进行盘库，可选择单个或多个仓库同时盘点
- **盘库状态**：
  - `待录入`：已发起盘库，等待录入盘点数据
  - `待复核`：已录入盘点数据，等待另一个仓管员复核
  - `已完成`：复核通过，盘库完成
  - `已驳回`：复核不通过，需要重新录入数据

### 盘库流程

#### 第一步：发起盘库（仓管员操作）
1. 登录系统后，进入「盘库管理」页面
2. 点击「发起盘库」按钮
3. 选择需要盘点的仓库
4. 填写盘库备注（可选）
5. 点击「确认发起」，系统自动生成盘库任务
6. 系统会自动拉取当前所选仓库的所有商品库存数据

#### 第二步：录入盘点数据（仓管员操作）
1. 在盘库列表中找到刚发起的盘库任务，状态为「待录入」
2. 点击「录入数据」按钮
3. 系统会显示该仓库下所有商品的**系统库存数量**
4. 仓管员根据实际盘点情况，录入**实际库存数量**
5. 系统会自动计算**盘盈/盘亏数量**和**盘盈/盘亏金额**
6. 所有商品录入完成后，点击「提交数据」
7. 盘库状态变为「待复核」，等待另一个仓管员复核

#### 第三步：盘库复核（另一个仓管员/管理员操作）
1. 由另一个不同的仓管员或管理员登录系统
2. 进入「盘库管理」页面，找到状态为「待复核」的盘库任务
3. 点击「复核」按钮
4. 核对盘点数据和系统库存数据的差异
5. 如有疑问，可以驳回并要求重新盘点，填写驳回原因
6. 如数据无误，点击「复核通过」
7. 盘库完成，系统自动更新库存数据，并生成盘库报表

#### 第四步：查看盘库报表
1. 盘库完成后，可在盘库列表中点击「查看详情」
2. 查看完整的盘库报表，包括：
   - 盘库基本信息（时间、仓库、操作人员）
   - 商品盘点明细（系统库存、实际库存、盘盈盘亏）
   - 盘盈盘亏汇总统计
   - 支持导出 Excel 报表

## ⚙️ 系统配置（管理员权限）

### 盘库频率配置
1. 管理员登录系统
2. 进入「系统设置」页面
3. 找到「盘库配置」选项
4. 可选择盘库频率：
   - 月度：每月一次
   - 季度：每季度一次（默认）
   - 半年度：每半年一次
   - 年度：每年一次
5. 可配置盘库提醒提前天数
6. 保存配置后生效

### 用户管理
1. 管理员登录系统
2. 进入「用户管理」页面
3. 支持新增、编辑、删除用户
4. 可分配用户角色：管理员/仓管员
5. 可重置用户密码

---

## 📊 数据查询

### 库存查询
- 支持按商品名称、SKU、分类、仓库等条件查询库存
- 实时显示当前库存数量、库存预警状态
- 支持导出库存数据

### 盘库记录查询
- 支持按时间范围、仓库、状态等条件查询历史盘库记录
- 可查看每个盘库任务的详细信息和报表
- 支持导出盘库记录

---

## 💡 操作注意事项

1. **盘库权限**：盘库录入和复核必须由不同的用户操作，同一用户不能同时进行录入和复核
2. **数据准确性**：录入盘点数据时请仔细核对，确保数据准确无误
3. **网络要求**：操作过程中请保持网络连接，避免数据丢失
4. **数据备份**：建议定期备份数据库，防止数据丢失
5. **密码安全**：首次登录后请及时修改默认密码，确保账号安全
